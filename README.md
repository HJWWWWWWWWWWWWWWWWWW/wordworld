# 斗破苍穹：沉浸式回合制 RPG

游戏不再以逐章阅读为核心，而是采用传统 RPG 主循环：

1. 在据点休息、修炼和整理行囊。
2. 前往已解锁区域探索，触发地图奇遇。
3. 在逐回合战斗中选择普攻、斗技、防御、丹药或逃跑。
4. 获取历练、战利品和冒险阅历，通过推进剧情开放新地图。
5. 消耗冒险阅历完成关键目标，具体过程由玩家自由决定。

## 剧情原则

- 小说仅用于提供世界背景、关键人物和关键冲突。
- 游戏不会展示原版小说全文，也不会要求逐章推进。
- 主线被重构为 38 个按章节实际顺序排列的 RPG 关键阶段，显式覆盖工作簿中的全部 21 个剧情里程碑与关键桥梁。
- 每个阶段包含“准备/进入”和“核心行动/决断”等子节点，完成子节点后才会结算阶段结果。
- 玩家可以自行决定如何修炼、探索、战斗、建立关系和获取资源。
- 只要满足关键目标，系统就会结算对应结果并进入下一阶段。
- 地图只随剧情阶段开放，等级与炼药等级不会提前解锁区域。
- 药老苏醒前，修炼只增加灵魂力量与冒险阅历，不会增加经验、斗气或境界等级。
- 战斗中可选择“本场自动战斗”；系统会根据双方生命、斗气、预计伤害和技能触发条件智能行动。
- 自动战斗会在低生命时吃药、斗气不足时普通攻击，并只在合适血量阶段施放终结类斗技。
- 战斗伤害、治疗与斗气恢复均按实际生效值结算，不显示溢出数值。

## 日期、昼夜与强制日程

- 每日分为清晨、午后、傍晚、深夜四个时段。
- 探索可选择随性游历、侦察地形、采集资源、追猎强敌或调查线索；不同取向会改变体力消耗、即时收益与遭遇偏好。
- 每次探索选择、遭遇处理和主动离开都会明确反馈行动意义、代价与收益。
- 探索、移动、修炼、切磋和推进主线会消耗一个时段。
- 夜间探索消耗更多体力，但会获得额外冒险阅历；深夜不能前往危险区域。
- 休息会推进到次日清晨并恢复生命与体力。
- 关键日程到时会强制中断自由行动，根据玩家此前准备情况结算奖励或后果。
- 首个节点为第 3 日清晨的萧家族内试炼，目标是达到等级 2 并赢得至少一次切磋。

## 已接入的 Excel 配置

- 38 个重新设计的关键剧情阶段及其 118 个子节点，原始章节仅保留为后台参考数据
- 134 张地图与 155 条地图奇遇
- 敌人、道具、斗技功法、人物关系、属性和境界成长
- 剧情驱动的地图开放、体力消耗、随机权重和战斗掉落

数据全部从 `story/text_game_event_schema_v4.xlsx` 动态读取，无需安装 `openpyxl`。

## 项目结构

```text
wordworld/
├─ src/wordworld/
│  ├─ core/           # 游戏规则与核心引擎
│  ├─ data/           # 剧情工作簿解析与数据加载
│  ├─ ui/             # 控制台交互界面
│  └─ config/         # 项目资源路径配置
├─ story/             # 剧情与游戏配置数据

```

## 运行

```powershell
# 命令行版（纯文本）
python run.py

# 画面版（pygame 瓦片地图 + 回合战斗）
python run_pygame.py

# 旧版 GUI（tkinter）
python run_gui.py
```

### Pygame 版操作

| 按键 | 功能 |
|------|------|
| 方向键 / WASD | 角色移动 |
| 空格 / 回车 | 交互（对话/开启宝箱） |
| R | 休息 |
| C | 修炼 |
| M | 菜单 |
| ESC | 退出 |

### 新增战斗机制

| 系统 | 说明 |
|------|------|
| **属性克制** | 五属性循环：火→风→毒→冰→雷→火。用克属攻击伤害 1.5x，可击破敌人护盾 |
| **意图预判** | 灵魂感知>=20 时显示敌人下回合行动（攻击/防御/斗技），支持策略决策 |
| **连击系统** | 连续攻击叠加连击数，每层+12%伤害（最高+120%），被技能击中则重置 |
| **蓄力系统** | 消耗一回合蓄力，恢复5斗气，下一击伤害x2.0（每场战斗限一次） |
