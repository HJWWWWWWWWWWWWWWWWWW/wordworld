# 系统审计报告（2026-06-12）

## 本轮已修复

1. 修复药鼎、异火等物品先消耗再执行失败的问题。
2. 纳戒已接入通用物品使用入口。
3. 未实现的物品效果会安全拒绝，不会消耗物品或残留部分效果。
4. 修复回城卷轴调用不存在方法导致崩溃的问题。
5. 修复旧存档计数式背包迁移时丢失同类物品数量的问题。
6. 修复拍卖行背包已满时先扣款、读取列表推进倒计时、刷新覆盖玩家上架物品的问题。
7. 功法斗气上限与六系抗性已进入实际结算。
8. 为敌人历史技能 ID 增加兼容映射，避免 Boss 和精英技能失去伤害加成。
9. 建立统一持续状态系统，支持回复、护盾、增益、减益、吸血、反伤和抗性。
10. 战斗物品已接入战斗行动，支持毒、控制、逃跑、护盾和复活。
11. 赠礼现在必须选择目标，控制台、TUI、GUI 与 Pygame UI 均已接入。
12. 按设计取消装备强化、修理、鉴定和宠物系统，并移除对应特殊道具、配方和掉落。

## 明确取消的特殊道具

- 修理锤、鉴定卷轴、魔兽食物、驯兽绳。
- 初级至神级强化石。

这些道具会从新生成数据、运行时道具表、旧存档背包、炼药配方和掉落池中排除。

## 剩余缺口

- 成长道具：修炼增益、炼药增益、经验增益、洗点、技能遗忘。
- 世界道具：强制或回避遭遇、藏宝图、秘境地图、天气修改。
- 功法属性：命中率、定身抗性和灵魂属性仍需要更完整的跨系统结算。

## 后续建议顺序

1. 接入成长道具，明确永久成长与临时增益的边界。
2. 接入世界道具，并统一地图、遭遇和天气状态。
3. 补全功法命中、控制抗性与灵魂属性的结算入口。
